# Labels Guide

## Naming Conventions
- Use lowercase, kebab-case labels (letters, digits, and hyphens).
- Allowed characters: letters, digits, `#`, `-`, `_`, `.`, `:`, `/`.
- No spaces or commas (labels are comma-separated in inputs).
- Use `dimension:value` for paired concepts (e.g. `state:patrol`, `priority:p1`).
- Prefer a single canonical spelling to avoid alias collisions.

## Best Practices
- Keep labels short, descriptive, and stable over time.
- Reuse existing labels before creating new ones.
- Group related labels under a shared dimension (team, area, state).
- Avoid one-off labels unless they add lasting value.
- Review label usage periodically and consolidate duplicates.

## Examples
- `bug`
- `ui`
- `backend`
- `state:patrol`
- `priority:p1`
- `team:infra`
